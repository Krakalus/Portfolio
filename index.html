Attested Logs: Tamper-Evident Black Box for Multi-Agent AI Conversations

Apart Research Technical AI Governance Challenge • Jan 30–Feb 1, 2026

  • Led full implementation of an open-source Python library for cryptographically attested logging of AI agent interactions — each message signed with Ed25519 by its agent, hash-chained (SHA-256 prev_hash), and anchored to trusted organizational public keys for offline verification of integrity, authenticity, and ordering.
  • Built core from scratch over an intense solo weekend: frozen dataclasses (Message, Proof, Certificate), key generation/signing, RFC 8785 deterministic JSON canonicalization, domain-separated hashing, linear chain logic, and offline LogVerifier.
  • Integrated with AutoGen (manual hooks) and LangGraph (callback handler); delivered runnable demos using real GPT-4o-mini conversations showing end-to-end signing, verification, and reliable tamper detection (signature and hash failures caught across chains of 2–4 messages).
  • Augments ECMA-430 (NLIP) multimodal message envelopes with embedded proofs and chain links — creates self-contained, forensic-grade audit trails for AI incidents (airplane black box parallel), EU AI Act compliance, and cross-organizational trust in high-stakes agent systems.
  • 20+ passing pytest cases covering crypto, chain integrity, cert handling, and tamper injection; MIT licensed, actively developed repo.
